Ruby Classes vs JavaScript Constructors

October 5, 2015

In my previous post about creating classes with Ruby, I gave an example of creating an attendance book. Today, I will continue with that example. I will show you a basic layout of adding information to a Ruby class and then introduce you to doing the same thing using JavaScript constructors.

class Attendance

def initialize(first_name, last_name, age)
@first_name = first_name,
@last_name = last_name,
@age = age
end

Here, we have a class called, "Attendance." There's an initialize method that has three arguments for the students' first names, last names and ages. Inside that method, these arguments have been assigned to instance variables so we could use them inside other methods we create. In JavaScript, we can do something similar that will also allow us to add students' information. We can do this using constructor functions: 

function student(firstName, lastName, age){
this.first = firstName;

this.age = age
}

This time, we created a function called, "student." It also takes in the same three arguments, this time in lowerCamelCase. Now, the syntax is different where the JS version of Ruby's 'end' is ending with a brace "}" (because the function opens one up). Inside those braces, we use 'this' to signify the function and followed by the argument, we can assign the argument to it (almost like an instance variable). Now, let's look at how we can add these arguments to a class vs a constructor. 

roll = Attendance.new("Cameron", "Quinn", 23)

var student1 = new student("Cameron", "Quinn", 23)

In both cases, we're adding a student's information. In a Ruby class, we would create other methods to make these students appear present or absent. In Javascript, we could create yet another function that determines being absent from being present to then use these functions in the variables we created for each separate student.
